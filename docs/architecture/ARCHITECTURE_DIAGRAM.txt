│                            KEY TECHNICAL DECISIONS                              │
└─────────────────────────────────────────────────────────────────────────────────┘

DECISION 1: Three Different Animation Durations
  2.5s (border) : 4.0s (shimmer) : 4.5s (glow)
  └─→ Creates interference pattern (LCM = 10.0s)
      └─→ Prevents perceived repetition for first 10 seconds
          └─→ User resets attention before pattern repeats
              └─→ Feels continuously novel and engaging

DECISION 2: Use Masks Instead of Alternatives
  ✓ mask-image (pixel-perfect visibility)
  ✗ overflow (can't create cone shapes)
  ✗ opacity (fades entire element)
  ✗ display (all-or-nothing visibility)
  └─→ Masks enable sophisticated visibility control
      └─→ Conic mask rotates with border
      └─→ Radial mask creates cone shape

DECISION 3: Dual-Layer Background
  Layer 1 (padding-box):  black interior
  Layer 2 (border-box):   animated gradient border
  └─→ Keeps animation localized to 2px border area
      └─→ Prevents gradient from overwhelming interior
          └─→ Maintains button readability

DECISION 4: Transform: translate() for Animation
  ✓ transform (GPU-accelerated, 60fps)
  ✗ top/left (CPU-intensive, drops frames)
  └─→ GPU acceleration ensures smooth performance
      └─→ Works well on mobile devices
          └─→ Minimal battery impact

DECISION 5: Linear Timing Function
  ✓ linear (constant speed, no easing)
  ✗ ease (variable speed, more CPU)
  └─→ Constant rotation feels mechanical and precise
      └─→ No easing in/out prevents pause perception
          └─→ Continuous motion feels active

┌─────────────────────────────────────────────────────────────────────────────────┐
│                         BROWSER RENDERING PIPELINE                              │
└─────────────────────────────────────────────────────────────────────────────────┘

1. PARSE (@property declarations)
   └─→ Register --gradient-angle as <angle> type
       └─→ Enable smooth interpolation engine

2. CALCULATE (Animation engine)
   └─→ Compute frame-by-frame values:
       ├─→ border-spin: --gradient-angle progresses 0° → 360°
       ├─→ shimmer: transform translates (-100%, -100%) → (100%, 100%)
       └─→ breathe: opacity oscillates 0.3 → 0.8 → 0.3

3. APPLY (CSS engine)
   ├─→ Update gradient starting points (conic-gradient)
   ├─→ Update mask visibility (conic-gradient mask)
   ├─→ Update element position (transform: translate)
   └─→ Update element opacity

4. RENDER (GPU acceleration)
   ├─→ GPU renders rotated gradient (border-box)
   ├─→ GPU renders masked sparkles (visibility calculation)
   ├─→ GPU renders translated pseudo-element (directional shimmer)
   └─→ GPU adjusts opacity (breathing glow)

5. COMPOSITE (Final output)
   └─→ Stack all 4 layers with z-index values
       └─→ Display to user as single button

Result: 60 FPS animation with minimal CPU overhead
        GPU-accelerated rendering = smooth motion
        Linear timing = predictable CPU load
